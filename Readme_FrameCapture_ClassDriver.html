1. Description

Frame Capture Example for Omnivision OV7692 sensor on the ADZS-BF707-BLIP2 Board.

2. Processor

  • ADSP-BF707

3. Overview

This example application demonstrates the capture of Video on the ADZS-BF707-BLIP2 board and writing it to a file on the PC using JTAG. The board has an Omnivision and Aptina sensor on it. This application demonstrates the Omnivision sensor functionality. The application drives the sensor to capture images and then sends it over JTAG to the PC. The output file omnivision_ov7692_capture.dat is generated in the project folder.

4. Hardware Requirement and Setup

ADZS-BF707-BLIP2, Board Rev 1.1, BOM Rev 1.1


No Switch and Jumper Settings are required.

 

         Connect a USB Micro-B cable to connector P4 on the board and to one of PCs USB ports on the other end to power the board.

         Connect the ICE-2000 to the 'JTAG'(P3) on the board. Connect a USB Mini-B cable to the USB port on ICE and the other end to one of the PC's USB ports.

 

5. Running the Example

1.     Be sure you have read the Hardware Requirement and Setup section of this file and followed their instructions.

2.     Open CrossCore Embedded Studio (CCES).

3.     Use the Example Browser to open this example project and build it. If you need help refer to the CCES documentation under section Browsing Examples and Code Sketches and Building Projects Manually.

4.     Create and launch a debug configuration which loads the executable. If you need help refer to the CCES documentation under section Creating a Launch Configuration for a CrossCore Embedded Studio Application.

Note:
The ADZS-BF707-BLIP2 Board is a custom board and hence to configure the reset values of the registers specific to the board, a custom board specific file needs to be provided. The custom board specific file for the ADZS-BF707-BLIP2 board is ADZS-BF707-BLIP2-proc.xml.

Follow below steps to use this file for running the application,

1.     While creating Debug Configuration, select Custom Board Support tab, and check the Enable customizations check box

2.     Click on Browse, and select the ADZS-BF707-BLIP2-proc.xml file present in project folder.

3.     Click on OK. In the Run/Debug settings click on Apply to save and apply the changes.

5.     Now run the executable by clicking Debug or by clicking Close and then pressing the F5 key.

6.     The example captures video frames and saves them as a .dat file in PC. A Raw video player can be used to display the captured video .dat file. The format of the images in the file is UYVY packed and the size is 640x480. Some video players show the format as UYVY or YUV422. Choose appropriately.
